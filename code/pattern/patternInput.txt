**Issue**: Gcc versions prior to 8.4.0 generate switch jump tables (indirect calls) in retpoline builds, creating speculative execution attack vectors that retpolines are meant to mitigate, while clang and newer gcc avoid this by default.

**Tag**: security hardening, compiler flag fix

**Purpose**: To add `-fno-jump-tables` to the build flags for gcc when retpolines are enabled, ensuring jump tables are not generated and eliminating unnecessary indirect jumps that could be exploited.
