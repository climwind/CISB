步骤二：识别编译器的假设与优化行为。
编译器基于C语言规范进行优化。C标准规定，对有符号整数进行移位，如果移位位数大于或等于类型的宽度，或者左移导致有符号溢出，则行为是未定义的。对于无符号整数，如果移位位数大于或等于类型的宽度，行为也是未定义的（C99 6.5.7p3）。因此，编译器假设程序中不会出现未定义行为。基于这个假设，编译器可以优化掉依赖于未定义行为的代码。例如，如果编译器看到“1 << s_log_groups_per_flex”，并假设s_log_groups_per_flex是一个合理的值（因为未定义行为不会发生），那么它可能优化掉后续的检查。具体来说，对于无符号类型，如果移位位数大于等于类型宽度，结果是未定义的，编译器可以假设这种情况不会发生，因此可能将“groups_per_flex == 0”的检查视为永远为假而消除。在提交消息中提到，Clang 3.0优化掉了“groups_per_flex == 0”的检查，因为对于无符号整数，1 << x（x在0到类型宽度-1之间）永远不会为0，所以groups_per_flex == 0永远不会成立，但这里x可能大于等于类型宽度，但编译器假设未定义行为不发生，所以认为x一定在0到宽度-1之间，因此1<<x不为0。这导致检查被部分消除。
